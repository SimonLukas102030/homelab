Local AI on the Homelab

LLMs (Llama 3, Mistral, Qwen), image gen (Stable Diffusion, Flux), Speech-to-Text (Whisper) — all local, no OpenAI API. GPU options, performance expectations, setup with Ollama + Open WebUI. What 24 GB VRAM can actually do.

01What you can run locally

With 24 GB VRAM (Tesla P40 or RTX 3090) you're in a comfortable class — you lose nothing compared to GPT-4o-mini or Claude Haiku for everyday tasks. Only the top frontier models (Claude Opus, GPT-5) remain cloud-only.

💡
Why Tesla P40 for the Medium tier: Best €/GB VRAM ratio on the market. €9 per GB VRAM (vs. RTX 3090 at €27/GB). Pascal architecture is slower per FLOP, but sufficient for LLM inference — especially since VRAM size matters more than speed (does the model fit? yes → usable). Cooling mod is standard, ~€20.
💡
Why RTX 3090 for the Pro tier: 3× faster than P40 in inference, 5–10× faster in SDXL/Flux. Plus: can double as a gaming card via Parsec/Sunshine. If you're serious about local AI, you eventually want FP16-capable hardware — the 3090 is the last card that will be supported for a long time.

03Setup — Ollama + Open WebUI

Recommended stack: Ollama as the inference engine (auto GPU detection, model management via CLI) + Open WebUI as the frontend (ChatGPT-style UI, multi-user capable).

Architecture

VM 110 ai-server (Ubuntu 24.04, GPU passthrough) │ ├── ollama (port 11434, OpenAI-compatible API) ├── open-webui (port 3000, Web UI) └── n8n / SillyTavern / etc. (optional, per Docker) GPU Passthrough: PCIe bus <Tesla P40 or RTX 3090> → VM 110 Model Storage: /var/lib/ollama/models (~50 GB for 5–6 models)

Installation

  1. Proxmox: enable GPU passthrough — enable IOMMU in BIOS, register vfio_pci for the GPU vendor IDs. Details below.
  2. Create VM: Ubuntu 24.04 Server, 8 vCores, 16 GB RAM, pass GPU through as PCI device. Boot order: virtio-disk first.
  3. NVIDIA drivers in guest: sudo apt install nvidia-driver-550 nvidia-utils-550. nvidia-smi must show the card.
  4. Install Ollama: curl -fsSL https://ollama.com/install.sh | sh. Auto-detects the GPU via nvidia-smi.
  5. First model: ollama pull llama3.1:8b. Then ollama run llama3.1:8b for a chat test.
  6. Open WebUI via Docker: docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui ghcr.io/open-webui/open-webui:main.
  7. NPM (see Software page) → ai.simonsstudios.de → 192.168.20.110:3000.

GPU Passthrough (Proxmox host)

# 1. IOMMU in kernel
echo "intel_iommu=on iommu=pt" >> /etc/default/grub
# (for AMD: amd_iommu=on iommu=pt)
update-grub

# 2. VFIO modules
echo "vfio
vfio_iommu_type1
vfio_pci
vfio_virqfd" >> /etc/modules

# 3. Find GPU IDs
lspci -nnv | grep -i nvidia
# Result: 03:00.0 ... [10de:1b06]
#         03:00.1 ... [10de:10f0]  (GPU audio component)

# 4. Bind VFIO to the IDs
echo "options vfio-pci ids=10de:1b06,10de:10f0" \
  > /etc/modprobe.d/vfio.conf

# 5. Blacklist NVIDIA driver on host
echo "blacklist nouveau
blacklist nvidia
blacklist nvidiafb" > /etc/modprobe.d/blacklist-nvidia.conf

# 6. Reboot
update-initramfs -u
reboot

# 7. Verify
dmesg | grep -i vfio
# Expected: vfio-pci bound to GPU

In the Proxmox web UI: VM → Hardware → Add → PCI Device → select the GPU, check "All Functions" + "PCI-Express".

Open WebUI — Highlights

  • Multi-model support — switch between Llama, Phi-4, Qwen via dropdown
  • RAG (Retrieval-Augmented Generation) — upload files, chat with them
  • Code interpreter with Python sandbox
  • Voice chat via Whisper + Piper TTS
  • Image gen integration — if you run AUTOMATIC1111 or ComfyUI alongside
  • Conversation branching like Anthropic Claude

04Additional Workloads

Stable Diffusion / SDXL — AUTOMATIC1111 or ComfyUI

SDXL is the go-to for local image generation. AUTOMATIC1111 is the classic UI with a plugin ecosystem (ControlNet, LoRAs). ComfyUI is node-based (workflow programming) — more modern, Flux-capable, but steeper learning curve. Both run in the same AI VM, port 7860 (A1111) or 8188 (Comfy).

Whisper — automatic subtitles

whisper.cpp on the GPU, Speech-to-Text for your Plex library — auto-generates subtitles for all movies and series. Run pattern: cron job, all new files in /mnt/media get subtitled overnight.

n8n — AI automation

Self-hosted Zapier alternative. Drag-and-drop workflows: "When new email in inbox → summarize with Llama → post to Discord webhook". With a local LLM endpoint (http://ollama:11434) the entire AI logic runs on-prem.

Coding Assistant — Continue.dev / Cody

VS Code plugin Continue.dev connects to your local Ollama endpoint. You get local code completion with Qwen 2.5 Coder 32B (fits in 24 GB) — near GPT-4o-level coding without API costs. Privacy advantage: no code leaves your network.

05Performance Tips

  • Choose quantization: Q4_K_M is the default recommendation — best quality-to-VRAM ratio. Q5_K_M if you have VRAM headroom, Q3 only when absolutely necessary (quality degrades noticeably).
  • Context size: don't set it to the maximum — 8K context in Llama 3 is enough for 99% of chat tasks. 32K context consumes extra VRAM and is often slower.
  • Batch size: set in Ollama via num_batch. Default 512 is fine. Lower to 128 if VRAM is tight.
  • Flash Attention (supported on RTX 30/40, not P40): OLLAMA_FLASH_ATTENTION=1 saves 20–40% VRAM on long context.
  • KV cache quantization: OLLAMA_KV_CACHE_TYPE=q8_0 halves KV cache memory. Helps with large contexts.
  • Model selection per task: Phi-4 14B for fast responses, Qwen 2.5 32B Coder for programming, Llama 70B-Q4 for deep reasoning. Switchable via dropdown in Open WebUI.

06Cost Comparison vs. Cloud API

Is local AI worth it compared to the OpenAI/Anthropic API? Quick math:

ScenarioLocal (Medium build)Cloud (GPT-4o-mini, ~$0.15/1M input + $0.60/1M output)
Casual chat (1k tokens/day) ~€0/day ~$0.0008/day
Heavy coding (50k tokens/day) ~€0/day (+electricity) ~$0.04/day = $1.20/month
Massive (500k tokens/day, pipelines) ~€0/day ~$0.40/day = $12/month
Image gen (50/day SDXL) ~€0 DALL-E: $0.04/img × 50 = $2/day
⚠
Reality check: Cloud APIs are dirt cheap for casual use. Your local AI setup will not pay off financially against GPT-4o-mini if you only do chat. Worth it for: privacy (your data stays local), unlimited usage without rate limits, custom fine-tunes, bulk image gen, RAG over large datasets, or simply because it's cool.
